# 语词汇释方法论 · 简牍/古文字工具书生成

## 适用范围

本 skill 用于把一批围绕同一批简牍材料（例如长沙五一广场东汉简牍、走马楼吴简、岳麓秦简、张家山汉简等）的研究论文，整理为一本按音序排列的「语词汇释」工具书。

**典型场景：**
- 用户有一批同一主题的学术 PDF 论文（例如某批简牍的考释论文）
- 用户希望按辞典/训诂体例汇集这些论文中讨论的语词
- 用户希望对同一字词，聚合多位学者的不同说法，按时序对照
- 用户提到"徐碩""语词汇释""音序排列""跨论文聚合"等关键词

**标杆范本：**
徐碩，《长沙五一广场东汉简牍（壹）-（肆）》语词汇释，吉林大学硕士学位论文，2023.

## 核心方法论

工具书生成是**跨论文聚合**而非**单论文逐条提取**。关键差别：

| | 单论文逐条提取（错误方向）| 跨论文聚合（正确方向）|
|---|---|---|
| 单位 | 论文 → 词条 | 词条 → 多论文 |
| 输出 | 每篇论文一组词条 | 每个字词一个辞条，下含 N 位学者解读 |
| 检索 | 难以查"同一字在几位学者那里怎么说" | 直接查到一个字下所有学者的不同说法 |

### 输入要求

1. **简牍原始材料目录**（必备）
   - 整理小组出版的简牍释读本（如《长沙五一广场东汉简牍选释》《长沙五一广场东汉简牍（壹）-（陆）》）
   - 用作简号体系基准

2. **研究论文 PDF 集合**（必备）
   - 围绕该批简牍的学术研究论文
   - 期刊论文、会议论文、学位论文、专著章节、网络文献都可
   - 数量参考：徐碩硕士论文收录了约 200 篇

3. **可选**：现有的语词卡片、术语清单（如有，可加快定位）

### 输出体例

成稿文档结构：

```
[标题页]

[摘要]（中文 + 英文双语）

[凡例]（核心！见下文凡例条款）

[目录]

[绪论]
  一、简牍出土与公布情况
  二、简牍性质
  三、专题研究综述（按子领域分节，如：历史地理、法制史、吏治、人口流动、
                   文书行政、社会生活、书迹研究、文字现象）

[正文]：A B C D ... Z 按音序排列的辞条

[读音未详]（不确定读音的辞条）

[附录]
  附录一、辞条索引
  附录二、新旧简号对照表
  附录三、缀合情况补表
  附录四、文书编联、集成成果统计表
  附录五、简牍所见地名整理表（按需调整）

[参考文献]
  专著
  学位论文
  期刊论文
  会议论文集
  专著、论文集中的析出文献
  报纸文章
  网络文献
```

## 凡例条款（必须执行）

以下条款来自徐碩论文「凡例」，是输出文档的硬性规范：

1. **释文、图版基准** — 所涉及的释文、图版均为已发表材料；未发表材料一般只保留简号。

2. **辞条、释文用繁体字**。

3. **辞条按音序排列**（A-Z，同字母下按音节细分如 an/ba/bai/cao 等）。

4. **辞条标注用「[字]」**形式（如 [安成]、[案]、[白]）。

5. **简号列于辞条下**，一般 1-3 条；含简号、形制（木牍/竹简/两行木简/封检/封泥匣/签牌）。

6. **释义按学者发表时间排序**，每条解读前用 `【作者+年份】` 简称（如【袁延勝、崔林2020】、【姚遠2015】、【缪愷然2020】），同一作者同年多篇用 A/B/C 区分。

7. **每条解读末尾标注页码**：`（XX 页）` 或 `（XX-XX 页）`。

8. **个别需补充或辨析的辞条**，用 `【碩按】` 形式（若复刻徐碩范式，可改为「【按】」或自定义按语标识）。

9. **文字约定：**
   - 通假字用「（）」标出正体（如 "祭" 之异体作 "祭"）
   - 异体字较重要的用「[ ]」标出正体
   - 字形结构差异较小者直接改为正体
   - 文字不可辨识用「□」表示
   - 文字残损漫漶用外加方框表示
   - 竹简残断疑问用「☑」表示
   - 文字有疑问用「（？）」表示

10. **简号体系**（以五一简为例，可类推到其他简牍）：
    - C = 长沙；W = 五一广场；J = 井窖；J1 = 一号窖；圆圈数字 = 堆积层
    - 例：CWJ1③：71-26 = 长沙五一广场一号窖第三层 71 号简的第 26 部分
    - 整理号优先于原始号；无整理号者用《选释》编号，其次用《简报》编号

11. **简称体系**：所有引用文献用 `【作者+年份】` 简称，置于辞条释义前；详细出处见参考文献。

## 执行流程（六步）

### 第一步：建立简号体系与简称表

输出文件：`papers_codebook.json`

```json
{
  "source_volumes": [
    {"abbrev": "【简报】", "title": "湖南长沙五一广场东汉简牍发掘简报", "year": 2013},
    {"abbrev": "【选释】", "title": "长沙五一广场东汉简牍选释", "year": 2015},
    {"abbrev": "【壹】", "title": "长沙五一广场东汉简牍（壹）", "year": 2018},
    ...
  ],
  "scholars": [
    {"abbrev": "【袁延勝、崔林2020】", "authors": "袁延勝、崔林", "year": 2020, 
     "title": "长沙五一广场东汉简牍中的户籍问题", "venue": "《简帛研究》二〇二〇秋冬卷"},
    {"abbrev": "【姚遠2015】", "authors": "姚遠", "year": 2015, ...},
    {"abbrev": "【缪愷然2020】", "authors": "缪愷然", "year": 2020, ...},
    ...
  ]
}
```

建立时需要：
- 列出输入论文集中每一篇的简称
- 同作者同年的多篇用 A/B/C 区分（按论文集发表先后或按页码先后）

### 第二步：逐篇阅读论文，抽取语词卡片

**核心原则：必須用 Read PDF 多模態精讀，禁止用 pdftotext + 正則匹配的自動化方式抽取卡片。**

> **失败案例（不要走的路）**：曾嘗試用 `pdftotext` 批量提取 125 篇 PDF 文本，再用正則匹配抽出 698 條辭條。結果：
> - 釋義破碎：捕到的句子是論文裡的引用、例句、過渡句，不是真正的「釋義」
> - 簡號被換行切斷：`CWJ1\n③：325-2-3` 變成兩段
> - 簡號附近的句子並不在解釋這個字（噪音率 > 50%）
> - 學者簡稱從文件名提取，大量「Unknown」
> - 古籍引證被截斷
>
> 結論：徐碩 800+ 辭條是**人工精讀 200 篇論文寫卡片**，Claude 復刻必須同樣用多模態精讀。

**对每篇论文 PDF（多模態精讀流程）：**

1. **Read PDF**：每次最多 20 頁，超出分批讀。讓 Claude 視覺看到頁面（保留繁體字、引證、頁碼、簡號完整形式）。

2. **理解上下文後寫卡片**：對論文裡明確「在解釋某個字/詞」的段落，提取：
   - **語詞原文**（嚴格保留論文中的用字，繁體）
   - **簡號**（含全形冒號、圓圈數字、整理號）
   - **形制**（木牘/竹簡/木兩行/封檢/封泥匣/簽牌）
   - **釋義內容**（300-1000 字，保留古籍引證的完整引文）
   - **頁碼**（論文裡該段落的頁碼）

3. **不立辭條的情況**：
   - 論文中只是引用例句，沒有給出對該字的解釋 → 跳過
   - 論文討論的是「文書結構」「案件流程」等宏觀問題，不在解釋具體字詞 → 跳過
   - 論文裡的「整理者注」「按語」等如果只是轉引，也要區分是否原創性釋義

**抽取格式**：JSONL，每行一張卡片
```jsonl
{"paper_abbrev": "【吳雪飛2017】", "term": "稽留", "code": "J1③：325-1-140",
 "shape": "木牍", "vol_num": "",
 "interpretation": "「稽留」為犯罪。《晉書·刑法志》：「興律有乏徭、稽留。」程樹德《漢律考》「興律」條下有「乏徭、稽留」...可見漢代存在「稽留」之罪。",
 "page": "192-193"}
{"paper_abbrev": "【繆愷然2020】", "term": "案", "code": "CWJ1③：71-26",
 "shape": "木牘", "vol_num": "《壹》257",
 "interpretation": "案，考察、審理。《史記·魏其武安侯列傳》：「丞相言灌夫家在潁川，橫甚，民苦之。請案。」《五一簡》中的「案」常常單獨使用，領起對案件原委的說明。",
 "page": "119-120"}
```

**精讀工作量參考**：
- 期刊論文（13-30 頁）：每篇精讀 + 寫卡片約 1-2 小時，每篇 5-15 張卡片
- 學位論文（300+ 頁）：聚焦於辭條相關章節（一般集中在某幾節），約 4-8 小時，30-100 張卡片
- 整體：200 篇論文約需 200-400 小時的精讀時間，分多次會話累積

### 第三步：按语词聚合，去重合并

合并规则：
- **聚合键** = 语词文本（不含通假符号、异体符号）
- **同一语词，多篇论文**：所有记录合并到一个辞条
- **简号列表去重**：同一辞条下的简号合并去重，按整理号顺序排列
- **解读按发表时间排序**

输出：`terms_aggregated.json`
```json
[
  {
    "term": "伍長",
    "pinyin": "wǔ zhǎng",
    "sort_key": "wuzhang",
    "codes": [
      {"abbrev": "《壹》", "num": "83", "shape": "木简", "id": "2010CWJ1①：87-1"}
    ],
    "interpretations": [
      {"paper_abbrev": "【袁延勝、崔林2020】", "text": "...", "page": "318"}
    ]
  },
  ...
]
```

### 第四步：拼音排序与音节分组

排序键：
1. 主排序：辞条首字拼音（不含声调）
2. 次排序：辞条全词拼音（含声调）
3. 末排序：原字 unicode

音节分组：
- 第一级：字母（A-Z）
- 第二级：音节（an/ang/ao/ba/bai/cao/ce 等）

排序输出：`sorted_entries.json` 按字母与音节分组

### 第五步：撰写绪论与附录

**绪论结构（必须包含）：**

1. **出土与公布情况**：来源、出土时间、出土地、整理出版进度
2. **简牍性质**：所属机构、文书类型分布、地域、时代上下限
3. **专题研究综述**：按学术领域分节，每节梳理学者贡献，文末用 `【作者+年份】` 标注引用
   - 典型子领域：历史地理、法制史、吏治、人口流动、文书行政、社会生活、书迹研究、文字现象

**附录五项**：
- 附录一：辞条索引（按音序列出所有辞条+所在页码）
- 附录二：新旧简号对照表
- 附录三：缀合情况补表
- 附录四：文书编联、集成成果统计表
- 附录五：地名整理表（按郡-县-乡-里整理，必要时附「今某省某市」对照）

### 第六步：排版生成 docx

使用 docx-js（npm `docx` 包），严格学术训诂体例：

- **字体**：宋体 / 楷体（标题）+ Times New Roman（西文/数字）
- **字号**：正文五号（10.5pt）；标题加大
- **行距**：1.5 倍
- **页面**：A4，上下 2.5cm，左右 2.5cm
- **页眉**：奇页论文标题，偶页章节名
- **页脚**：页码
- **章节分页**：每章节起始页另起
- **辞条排版**：
  - 辞条标题（[字]）独立成段，加粗
  - 「编号：」一行
  - 简号列表（每个简号一行）
  - 学者解读（每位学者起新段，开头 `【作者+年份】`）
  - 【按】末尾，若有

## 模式化辞条范式

每个辞条严格按以下结构：

```
[辭條]

編號：
《選釋》XX  形制 CWJ1③：XX-XX  （簡文示例，加粗辭條字）
《壹》XX    形制 CWJ1①：XX-XX

【作者A+年份】釋義內容...（XX 頁）
【作者B+年份】另一種釋義...（XX 頁）
【作者C+年份】補充見解...（XX-XX 頁）

【按】編者按語（如有需要補充或辨析）

```

### 实例（来自徐碩论文 p.31-32）

```
[安成]
編號：
《選釋》50  木兩行 CWJ1③：325-1-105

【選釋】安成，縣名《後漢書·郡國志》作"安城"，屬長沙郡。


[安民史]
編號：
《壹》57  竹簡 CWJ1①：25-26

【袁延勝、崔林2020】由於流民問題較為突出，因此政府一方面下詔同意著籍流入地外，
另外可能還設置"安民史"來安撫流動之人。簡文載：
    左戶曹史麟白，民自言辭如牒，請記告安民□  （《壹》54）
    府告安民史意，民自言□              （《壹》57）
（《壹》簡 54"安民"後面很可能是"史"，儘管簡文內容不多，但從"民自言""府告"
等詞語來看，"安民史"可能就是為招撫流民而設置。（320 頁）

【崔京2020】安民，從字面意義上來說，應是安撫民眾的意思。由以上兩處簡牘（《壹》54、
《壹》57）推測，安民史當是在處理司法行政案件中官民連接人，協助縣級官吏向涉及案件
的民眾準確傳達相關行政命令。（40-41 頁）

【姚遠2021】從兩漢史料及出土文獻資料中皆不見"安民史"，但《三國志·吳書》載：
"孫綝字子通，與峻同祖。琳父綽爲安民都尉。"其中可見"安民都尉"官職名稱，但無詳
細執掌。從職官名稱上看，"安民"應當以安撫百姓、穩定治安爲核心任務。
此外，五一簡中有很多文獻資料中少見的官職名稱，如"趣租史""丈田史"等...
（331-332 頁）
```

## 工具清单

执行本方法论需要：

1. **Read PDF**：多模态 PDF 阅读（识别繁体字、古文、引证、脚注）
2. **Python**：数据处理（JSON 聚合、排序、去重）
3. **pypinyin**：自动获取汉字拼音（`pip install pypinyin`）
4. **docx-js**：Word 排版（`npm install -g docx`）
5. **pandoc**（可选）：辅助文档转换

## 常见陷阱

1. **不要从单篇论文逐条提取**：徐碩范式是跨论文聚合，每个辞条下有多位学者按时序排列的解读。

2. **不要遗漏「（壹·N）」等整理号索引**：徐碩在辞条标题下标注《选释》《壹》-《陆》中的整理号，便于学者交叉查阅。

3. **简号体系不要混用**：CWJ1③：71-26（采集号）和《壹》257（整理号）是不同体系，要明确标注。

4. **【作者+年份】简称要前后一致**：同一作者同年多篇用 A/B/C 区分，建立简称表后整本统一。

5. **释义里的内部引用也要保留**：徐碩转引学者的话时会注明「转引自《XX》」或在脚注里给出原文出处。

6. **绪论的专题研究综述不能省**：506 页论文有 30 页绪论，是学术工具书的标配。

7. **辞条标题用全角方括号「[字]」**，不要用半角 `[字]`。

8. **音序排列必须用拼音**，不能用笔画。

## 验收标准

成稿应通过以下检查：

- [ ] 凡例完整，11 条规则全部落实
- [ ] 辞条按音序排列，A-Z 完整
- [ ] 每辞条下有 1-3 个简号 + 形制
- [ ] 每辞条下至少 1 位学者解读，多者按发表时间排序
- [ ] 每条解读以 `【作者+年份】` 开头，以 `（XX 页）` 结尾
- [ ] 辞条与正文均为繁体字
- [ ] 绪论含「出土公布」「简牍性质」「专题研究」三节
